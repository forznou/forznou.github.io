计算机网络

发表于 2023-06-10 更新于 2023-06-12
本文字数： 1.8k 阅读时长 ≈ 7 分钟

计算及网络内容有点多决定专门写篇博客，去整理一下

概述

互联网的基本特点：连通性和共享

世界上第一个计算机网络是ARPANET

互联网服务提供者ISP，例如中国的联通，移动，电信。

计算机网络的定义

定义：计算机网络是将地理位置不同的独立计算机系统，通过传输介质链接起来，它们遵守共同的协议，

目的：通信和资源共享。

计算机网络（简称网络）有若干节点和连接这些节点的链路组成

通信系统三个基本要素：信源，通信媒体，信宿。

按照网络的作用范围进行分类，广域网WAN,城域网MAN，局域网LAN，个人区域网PAN。

计算机网络传输中常用的三种有限传播媒介是：双绞线，同轴电缆，光纤。

网络协议的主要要素为，语法，语义，同步。

网络中时延主要是由传播时延，发送时延，和排队时延组成。

网络地址

MAC地址：数据链路层使用, 也叫物理地址: 绑定在网卡上的一个地址(硬件地址)，MAC 地址是”绝对唯一”。

IP地址：在网络层，一个网络内部的IP地址，不应该重复。用于标识网络主机、其他网络设备（如路由器）的网络地址。

端口： 在网络通信中，IP地址用于标识主机网络地址，端口号可以标识主机中发送数据、接收数据的进程。简单说：端口号用于定位主机中的进程。

格式:
 端口号是0~65535范围的数字，在网络通信中，进程可以通过绑定一个端口号，来发送及接收网络数据。

 两个不同的进程，不能绑定同一个端口号，但一个进程可以绑定多个端口号。

MAC地址是数据链路层使用地址，而IP地址是网络层和以上各层使用的地址，是一种逻辑地址

网络协议 物理设施 单位
应用层 http,dns,FTP，SMTP,SNMP,POP3,TELENT 网关 报文，消息
传输层 tcp,udp 报文，段
网络层 ip,ARP,ICMP,IGMP 路由器 数据包
数据链路层 PPP,ethernet 交换机网桥网卡 数据帧
物理层 集线器，中继器，放大器 比特

物理层

数据通信系统可分为三大部分：源系统，传输系统，目的系统

信道一般是用来表示向某一个方向传送的信息的媒体。

从通信双方信息交互方式来看，有三种基本方式，

单向通信，双向交替通信，双向同时通信。

传输媒体可分为两大类，引导性传输媒体（双绞线，同轴电缆，光纤），和非引导型传输媒体（无线，红外，大气激光）

常用的信道复分技术有：频分复用，时分复用，统计时分复用，码分复用，和波分复用。

光纤的优点是

  1. 传输耗损小，中继时间长，对远距离传输，特变经济。
  2. 抗雷电和电磁干扰性能好。
  3. 无串音干扰，保密性好，也不容易被窃听或截取数据。
  4. 体积小，重量轻。

数据链路层

数据链路层最基本的服务是将源计算机网络层来的数据可靠的传输到相邻节点的目标计算机的网络层。为达到这一目的，数据链路层必须具备一系列相应的功能，主要有：如何将数据组合成数据块（在数据链路层中将这种数据块称为帧，帧是数据链路层的传送单位）；如何控制帧在物理信道上的传输，包括如何处理传输差错，如何调节发送速率以使之与接收方相匹配；在两个网路实体之间提供数据链路通路的建立、维持和释放管理。

数据链路层使用的信道主要有点对点信道和广播信道两种。

数据链路层的三个基本问题是：封装成帧，透明传输，和差错检测。

PPP：点对点协议

简单；只检测差错，而不是纠错差错；不使用序号，也不进行流量控制；可同时支持多种网络层协议

适配器的的功能，包含了数据链路层以及物理层两个层次的功能。

Ethernet：以太网协议

以太网的硬件地址，即mac地址实际上就是适配器地址或适配器标识符 ，与主机所在的地址无关。源地址和目标地址都是48位长。

网络层

网络层提供的两种服务：面向连接的虚电路服务，无连接的数据报服务

网络层不提供服务质量的承诺。

ARP协议

地址解析协议ARP把IP地址解析为mac地址，它解决了同一个局域网上的主机或路由器的IP地址和MAC地址的映射问题。ARP的高速缓存可以大大减少网络上的通信量。

IP地址32位二进制代码，常用点分十进制记忆法。

ICMP协议

使用icmp并非为了实现可靠传输，ICMP允许主机或路由器报告差错情况和提供有关异常情况的报告。

ICMP报文的种类有两种，ICMP的差错报文和ICMP的询问报文。

ICMP的一个重要应用就是分组网间探测ping，用来测试两台主机之间的连通性。

IPV6

IPv6的目的地址可以是以下三种基本类型地址之一：单播，多播，和任播。

IPv6所带来的变化，

  1. 更大的地址空间（采用128位的地址）
  2. 灵活的首部格式
  3. 改进的选项
  4. 支持即插即用
  5. 支持资源的预分配
  6. IPV6的首部改为八字节对齐

IPv 6的地址采用冒号十六进制记法。

运输层

运输层向它上面的应用层提供通信服务

与网络层的区别

网络层为主机之间的通信提供服务，而运输层则在网络层的基础上，为应用进程之间提供服务。

UDP协议：用户数据报协议

  1. 无连接
  2. 尽最大努力交付
  3. 面向报文
  4. 无拥塞控制
  5. 支持一对一，一对多，多对一和多对多的交互通信
  6. 首部开销小（只有四个字段，源端口，目的端口，长度和校验和）

TCP：传输控制协议

三次握手四次挥手

  1. 面向连接
  2. 每一条tcp连接只能是点对点的
  3. 提供可靠交付的服务
  4. 提供全双工通信
  5. 面向字节流

应用层

应用层提供应用进程间的逻辑通信

HTTP：超文本传输协议
DNS：域名解析协议
FTP：文件传送协议
SMTP：简单邮件传送协议
SNMP：简单网络管理协议
POP3：邮件读取协议
Telnet：远程终端协议
