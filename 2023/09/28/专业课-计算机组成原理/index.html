各个硬件的工作原理1.2_2

主存储器

存储体 由记忆元件组成

MAR (memory address register)储存地址寄存器，存储地址

MDR (memory Data Register) 存储数据寄存器，存放实际数据

存储单元： 每个储存单元存放一串二进制代码

存储字 存储单元中二进制代码的组合

存储字长：在储存单元中的二进制代码的位数

储存元： 即储存二进制的电子原件，每个储存元可存 1bit

mar 位数反应存储单元的位数

mdr 位数 =存储字长

例如：mar 4 位 总共由2**4 个储存单元

mdr 16 位 每个储存单元可存放16 bit 一个字word =16 bit

易混淆 1个字节 Byte =8bit

1B=1个字节 1b=1个bit

大B 和小b 不一样

运算器的基本组成

ACC : 累加器，,用于存放操作数，或运算结果

MQ: 乘商寄存器，在乘除运算时，用于存放操作数或运算结果。

x: 通用的操作数寄存器，用于存放操作数

ALU : 算数逻辑单元，通过内部复杂的电路实现算数运算，逻辑运算

ALU 为核心部件

控制器的基本组成

CU （Control Unit）控制单元，分析指令，给出控制信号

IR (Instruction,Register): 指令寄存器，存放 当前执行的指令

PC(Program Counter): 程序计数器，存放下一条指令地址，有自动加一功能

完成一条指令 PC 取指令 IR 分析指令 CU 执行指令

计算机的性能指标1.3

MAR 位数反映存储单元的个数（最多支持多少个）

MDR 位数=储存字长=每个储存单元的大小

总容量=储存单元个数*储存字长 bit=储存单元个数*存储字长/8 Byte

例如：MAR为32位，MDR为8位

总容量=2**32*8bit=4GB

k:2**10,M:2**20,G:2**30,T:2**40

CPU 主频：CPU内数字脉冲信号震荡的频率

CLK ，每次，即CPU ,时钟周期

CPI（Clock cycle Per Instruction） : 执行一个指令所需的时钟数

CPU主频（时钟频率）=1/CPU时钟周期，单位为HZ,例子10HZ 每秒钟有10个脉冲信号

例题:某CPU主频为1000Hz，某程序包含100条指令，平均来看指令的CPI=3。该程序在该CPU上执行需要多久?

0.3秒

IPS(Instructions Per Second): 每秒执行多少条指令 IPS=主频/平均CPI

FLOPS(Floting-point Operationn Per Second): 每秒执行多少次浮点运算

数据通路带宽：数据总线一次所能并行传送的信息位数（各硬件部件通过数据总线传输数据）

吞吐量：指系统在单位时间内处理请求的数量

它取决于信息能多块地输入内存，CPU能多块地取指令，数据能多块地从内存取出或存入，以及所得记过能多快地从内存送给一台外部设备。这些步骤的每一步都关系到主存，因此，系统吞吐量主要取取决于主存的存取周期

响应时间：指从用户向计算机发送一个请求，到系统对该请求做出相应并获得它所需要得结果的等待时间。

通常包括cpu时间，（运行一个程序所花费的时间）与等待时间（用于磁盘访问，存储器访问，I/O操作，操作系统开销等时间）。

基准程序：是用来测量计算机处理速度的一种使用程序，以便于被测量的计算机性能可以与运行相同程序的其他计算机性能进行比较。

8421码，余三码，2421码。

8421码计算，如果计算结果落在了非法区间，就+6即0110

余三码：8421码+（0011）

2.1.3无符号整数的表示和运算

无符号整数，即自然数。

无符号整数在计算机内部的表示

通用寄存器只能存8位

计算机硬件如何做无符号整数的减法：

1.被减数不变，减数全部位按位取反，末尾+1，减数变加法

2.从最低为开始，按位相加，并往更高位进位

2.1_4带符号整数的表示和运算_原反补

原码，补码，快速转换

正数 原码 不变 转 补码

负数 原码 从左往右找到第一个1，这个1 左边的所有数值位 按位取反 ,转为补码

cache

cache(高速缓冲存储器)中的数据就是主存的一个子集，CPU访问存储器时会先访问cache，如果cache没有找到需要的数据，就会去主存找

Cache 的作用 :Cache的出现使得cpu可以不直接访问主存，而是和高速Cache 交换信息

Cache 命中率

cpu欲访问的信息已在Cache 中的比率

将主存的储存空间分块，贮存和cache 之间以块为单位进行数据交换

Cache和主存的映射方式

全相联映射

主存块可以放在Cache 的任意位置

直接映射

每个主存块只能放到一个特定的位置

Cache块号=主存块号%Cache总块数

组相联映射

Cache块分为若干组，每个主存块可放到特定分组的任意一个位置组好=主存块号%分组数
